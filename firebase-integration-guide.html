🔥 Firebase Integration Guide

RICA × Firebase

Step-by-step guide to replace localStorage with Firebase Auth + Firestore — multi-device, real-time, secure.

1
What Gets Replaced
Old vs. New architecture
FeatureBefore (localStorage)After (Firebase)
Login Hardcoded password in browser Firebase Auth — email/password
Registration Stored in localStorage (device only) Firebase Auth + Firestore profile
Orders / Inventory localStorage (device only) Firestore — /users/{uid}/data/orders
Activity Logs localStorage array Firestore — /users/{uid}/logs
Multi-device ❌ Not possible ✅ Works everywhere
Password reset Manual security question only Firebase email reset link
2
Create Your Firebase Project
Takes about 5 minutes — free tier is enough
  • Go to console.firebase.google.com → Add project
  • Name it rica-app (or anything you want)
  • Disable Google Analytics (not needed) → Create project
  • Click Authentication → Get started → Email/Password → Enable → Save
  • Click Firestore Database → Create database → Production mode → Choose a region close to your users → Enable
  • Click Project Settings (gear icon) → Your apps → click </> (Web) → Register app → copy the firebaseConfig object
💡 The free Spark plan includes 1 GB storage, 50K reads/day, 20K writes/day — more than enough for a small business tool.
3
Add the Script to Your HTML File
ONE line addition to RICA_rebuild.html

Open your RICA_rebuild__5_.html file and find the </head> closing tag. Add this line just before it:

<!-- Firebase Cloud Sync — add just before </head> -->
<script src="firebase-rica.js"></script>
⚠️ Place firebase-rica.js in the same folder as your HTML file. Both must be uploaded to GitHub together.

Then fill in your Firebase config at the top of firebase-rica.js:

const RICA_FIREBASE_CONFIG = {
  apiKey:            "AIzaSyABC123...",          // ← paste from Firebase Console
  authDomain:        "rica-app.firebaseapp.com",
  projectId:         "rica-app",
  storageBucket:     "rica-app.appspot.com",
  messagingSenderId: "123456789",
  appId:             "1:123456789:web:abcdef"
};
4
Update the Login Form (Username → Email)
Firebase uses email addresses, not usernames

In your HTML, find the login username input and change the label + placeholder:

<!-- FIND this in your HTML (~line 1771): -->
<label class="login-label">Username</label>
<input type="text" id="login-username" placeholder="Username" ...>

<!-- REPLACE with: -->
<label class="login-label">Email Address</label>
<input type="email" id="login-username"
       placeholder="your@email.com"
       autocomplete="email"
       onkeydown="if(event.key==='Enter')document.getElementById('login-password').focus()">

Also add a "Forgot Password?" link that uses Firebase reset email:

<!-- Replace the existing forgot password span with: -->
<span onclick="doForgotPassword()"
      style="font-size:12px;color:#888;cursor:pointer;text-decoration:underline;">
  Forgot your password?
</span>

<!-- Add this JS somewhere in your script section: -->
<script>
async function doForgotPassword() {
  var email = prompt('Enter your email address:');
  if (!email) return;
  try {
    await firebase.auth().sendPasswordResetEmail(email);
    alert('✓ Password reset email sent! Check your inbox.');
  } catch(e) {
    alert('Error: ' + e.message);
  }
}
</script>
5
Set Firestore Security Rules
IMPORTANT — prevents other users from reading your data

In Firebase Console → Firestore Database → Rules tab, paste:

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    // Each user can ONLY read/write their own documents
    match /users/{userId} {
      allow read, write: if request.auth != null
                         && request.auth.uid == userId;

      match /data/{table} {
        allow read, write: if request.auth != null
                           && request.auth.uid == userId;
      }
      match /logs/{doc} {
        allow read, write: if request.auth != null
                           && request.auth.uid == userId;
      }
    }
  }
}

Click Publish to apply.

🔒 Without these rules, your data is public! Always set rules before going live.
6
Deploy to GitHub Pages
Add firebase-rica.js to your repository
  • Add firebase-rica.js to your GitHub repository (same folder as the HTML)
  • Commit & push both files
  • GitHub Pages will serve both automatically

Add your GitHub Pages domain to Firebase Auth:

In Firebase Console → Authentication → Settings → Authorized domains → Add Domain → enter yourusername.github.io

✅ Firebase Auth works perfectly on GitHub Pages — no server needed! HTTPS is already enabled on github.io domains.

Your Firestore data structure will look like this:

Firestore collections:

users/
  {uid}/                          ← one document per user

      finance   → { records: [...] }
      products  → { records: [...] }

    logs/                         ← subcollection
      activity  → { logs: [...] }
7
Test the Integration
Verify everything works before going live
  • Open browser console (F12) — you should see RICA Firebase integration loaded ✓
  • Register a new account using the signup form (use a real email)
  • Check Firebase Console → Authentication → Users — your account should appear
  • Add an order in RICA, then check Firestore Console → users → your UID → data → orders
  • Open the same URL in a different browser / device, log in with the same email — the data should appear!
🔧
Common issues:
• "Firebase not defined" → make sure the <script> tag is correct and the file exists
• "auth/unauthorized-domain" → add your domain in Firebase Auth → Settings → Authorized domains
• "Missing permissions" → check that Firestore Rules are published correctly
• Data not loading → open DevTools → Network tab and check for failed Firestore requests

Optional Improvements
Enhancements you can add later

Real-time sync (live updates across devices):

// Add to firebase-rica.js after login — listens for changes in real time
function watchTable(table, callback) {
  userRoot().collection('data').doc(table)
    .onSnapshot(function(snap) {
      var records = snap.exists ? (snap.data().records || []) : [];
      _cloudCache[table] = records;
      callback(records);
    });
}

Export / backup data to JSON:

async function exportAllData() {
  var tables = ['orders', 'inventory', 'finance', 'products'];
  var backup = {};
  for (var t of tables) {
    backup[t] = CloudDB.getAll(t);
  }
  var blob = new Blob([JSON.stringify(backup, null, 2)], { type: 'application/json' });
  var a = document.createElement('a');
  a.href = URL.createObjectURL(blob);
  a.download = 'rica-backup-' + new Date().toISOString().slice(0,10) + '.json';
  a.click();
}

Migrate existing localStorage data to Firebase (one-time):

async function migrateFromLocalStorage() {
  var uid = firebase.auth().currentUser?.uid;
  if (!uid) { alert('Please log in first'); return; }
  var prefix = 'rica_' + (localStorage.getItem('rica_active_user') || 'default') + '_';
  var tables = ['orders', 'inventory', 'finance', 'products', 'kpis', 'pricing'];
  for (var t of tables) {
    var raw = localStorage.getItem(prefix + t);
    if (raw) {
      await CloudDB.saveAll(t, JSON.parse(raw));
      console.log('Migrated:', t);
    }
  }
  alert('✓ Migration complete! Your data is now in Firebase.');
}
